  What Was Done                                                                                                                                
                                                                                                                                               
  1. Fixed FlashInfer kernel compilation - PyTorch's cpp_extension was disabling half-precision operators. Fixed by adding to setup.py:        
  "-U__CUDA_NO_HALF_OPERATORS__",                                                                                                              
  "-U__CUDA_NO_HALF_CONVERSIONS__",                                                                                                            
  "-U__CUDA_NO_BFLOAT16_CONVERSIONS__",                                                                                                        
  "-U__CUDA_NO_HALF2_OPERATORS__",                                                                                                             
  2. Added dtype check in sdpa_prefill.cu and sdpa_decode.cu to skip FlashInfer for float32 (only supports 16-bit types):                      
  if constexpr (sizeof(DTypeQ) != 2 || sizeof(DTypeKV) != 2) {                                                                                 
      return cudaErrorNotSupported;                                                                                                            
  }                                                                                                                                            
  3. Rebuilt the extension: python setup.py build_ext --inplace
